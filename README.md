# Attacker System — Руководство по установке

Распределённая система атак: **1 командер** (atk-1) управляет **до 6 воркеров** (atk-2…atk-7).  
Все атаки запускаются с одной страницы в браузере.

---

## Что нужно перед началом

- 7 виртуальных машин Ubuntu 22.04 в одной сети
- SSH-доступ к каждой машине
- Defender VM уже запущен (`bash install.sh` выполнен)
- Знаешь **внутренний IP** defender-VM (посмотри командой `hostname -I` на той машине)

---

## Схема работы

```
Ноутбук (браузер)
       │
       ▼
  atk-1:5000/ui  ←─── ты управляешь отсюда
       │
       ├──push──▶  atk-2:5001  (worker)
       ├──push──▶  atk-3:5001  (worker)
       ├──push──▶  atk-4:5001  (worker)
       ├──push──▶  atk-5:5001  (worker)
       ├──push──▶  atk-6:5001  (worker)
       └──push──▶  atk-7:5001  (worker)
```

**Воркерам не нужен IP командера** — они просто ждут команд на порту 5001.  
Командер сам рассылает атаки всем добавленным нодам одновременно.

---

## Шаг 1 — Установить командер на atk-1

Заходишь по SSH на **atk-1** и запускаешь:

```bash
DEFENDER_IP=<внутренний IP defender> bash <(curl -s https://raw.githubusercontent.com/wester11/vkr_attacker/main/install_attacker.sh)
```

**Пример** (если внутренний IP defender — `10.129.0.5`):

```bash
DEFENDER_IP=10.129.0.5 bash <(curl -s https://raw.githubusercontent.com/wester11/vkr_attacker/main/install_attacker.sh)
```

Скрипт (~3-4 минуты) установит Python, инструменты атак, создаст systemd-сервис и запустит командер.  
В конце выведет:

```
╔══════════════════════════════════════════════════════╗
║              COMMANDER ГОТОВ                         ║
╚══════════════════════════════════════════════════════╝

  🎮 Веб-интерфейс:  http://89.169.X.X:5000/ui
  📡 API статус:     http://89.169.X.X:5000/status
```

> Запомни публичный IP atk-1 — через него открывается UI.

---

## Шаг 2 — Установить воркеры на atk-2 … atk-7

Заходишь по SSH на каждую ноду и запускаешь **одну команду без параметров**:

```bash
bash <(curl -s https://raw.githubusercontent.com/wester11/vkr_attacker/main/install_worker.sh)
```

Скрипт установит всё автоматически и в конце выведет **внутренний IP ноды**:

```
╔══════════════════════════════════════════════════════╗
║              WORKER ГОТОВ                            ║
╚══════════════════════════════════════════════════════╝

  📡 Этот воркер слушает на: 10.129.0.3:5001

  Следующий шаг:
  Открой UI командера → поле «Добавить воркера» → введи: 10.129.0.3
```

**Скопируй этот IP** — он понадобится на следующем шаге.

### Установить всех воркеров одновременно (с ноутбука)

Если не хочется заходить на каждую ноду по очереди:

```bash
for i in 2 3 4 5 6 7; do
  ssh root@atk-${i} \
    "bash <(curl -s https://raw.githubusercontent.com/wester11/vkr_attacker/main/install_worker.sh)" &
done
wait
echo "Все воркеры готовы"
```

> Все 6 устанавливаются параллельно — займёт ~3-4 минуты.

---

## Шаг 3 — Добавить воркеры в командер

1. Открой в браузере: **`http://<публичный IP atk-1>:5000/ui`**

2. В поле **«IP воркера»** введи **внутренний** IP воркера из шага 2 (например `10.129.0.3`)

3. Нажми **«Добавить воркера»**

4. В списке появится нода со статусом `online` (зелёная точка)

5. Повтори для каждого из atk-2…atk-7

> Если статус `offline` — воркер ещё не запустился. Подожди 10 секунд и обнови страницу.

---

## Шаг 4 — Запустить атаку

1. В поле **«IP цели»** введи **внутренний** IP defender-VM (тот же что указывал в Шаге 1)

2. Нажми кнопку нужной атаки — командер **мгновенно рассылает команду всем** `online` воркерам

3. Чтобы остановить — нажми **⛔ СТОП**

---

## Типы атак

| Кнопка | Тип | Что делает |
|--------|-----|-----------|
| 🌊 HTTP Flood | `flood` | `ab` — 500 000 GET-запросов, 200 потоков с каждой ноды |
| 💥 DDoS Spoof | `ddos` | 20 потоков Python, каждый запрос с рандомным X-Forwarded-For |
| 🔍 Nikto Scan | `scan` | Сканирование уязвимостей nikto, 120 сек |
| 🔑 Brute Force | `brute` | POST /login — 20 req/s со случайными логин/паролями |
| 💉 SQL Inject | `sqli` | 15 SQLi-пейлоадов на 4 эндпоинта, 10 req/s |
| 🐢 Slowloris | `slowloris` | 200 висячих TCP-соединений (исчерпывает пул nginx) |
| ⚡ Flash Crowd | `flash` | `wrk` — резкий легитимный всплеск (4 потока, 50 конн.) |
| 🌙 Медл. флуд | `slow` | `wrk` — медленный фоновый флуд /search |

## Проверка статуса

```bash
# Сколько воркеров подключено и текущая атака
curl http://<IP atk-1>:5000/status

# Список воркеров с их статусами
curl http://<IP atk-1>:5000/workers
```

Или просто смотри в UI — обновляется каждые 3 секунды.

---

## Управление через curl (без UI)

```bash
CMD="http://<IP atk-1>:5000"

# Установить цель
curl -X POST $CMD/target \
  -H 'Content-Type: application/json' \
  -d '{"target": "10.129.0.5"}'

# Запустить атаку
curl -X POST $CMD/attack/start \
  -H 'Content-Type: application/json' \
  -d '{"attack_type": "flood"}'

# Остановить всё
curl -X POST $CMD/attack/stop

# Добавить воркера
curl -X POST $CMD/workers/add \
  -H 'Content-Type: application/json' \
  -d '{"ip": "10.129.0.3"}'
```

## Диагностика

**Командер не запускается:**
```bash
sudo systemctl status commander
sudo journalctl -u commander -n 30
```

**Воркер не запускается:**
```bash
sudo systemctl status attacker-worker
sudo journalctl -u attacker-worker -n 30
```

**Воркер показывает `offline` в UI:**
- Проверь что внутренний IP правильный (не публичный)
- Порт 5001 открыт: `curl http://<IP воркера>:5001/status`

## Порядок установки — итого

| # | Где | Команда | Результат |
|---|-----|---------|-----------|
| 1 | atk-1 (SSH) | `DEFENDER_IP=... bash <(curl -s .../install_attacker.sh)` | Командер на :5000 |
| 2 | atk-2…7 (SSH) | `bash <(curl -s .../install_worker.sh)` | Воркер на :5001, выводит свой IP |
| 3 | Браузер | Открыть `http://atk-1:5000/ui` | Веб-интерфейс |
| 4 | UI | Добавить IP каждого воркера | Статус `online` |
| 5 | UI | Нажать кнопку атаки | Все ноды атакуют одновременно |
| 6 | UI | Нажать ⛔ СТОП | Все ноды останавливаются |
